# Orbytech

[![Stellar Wave](https://img.shields.io/badge/Stellar-Wave%20Project-blue)](https://www.drips.network/wave/stellar)

Orbytech is a Stellar-native development ecosystem focused on building modular, open-source infrastructure for decentralized payments, smart contracts, and developer tooling on Soroban.

## Overview

Orbytech is building a monorepo-driven blockchain stack designed to simplify how developers create, test, and deploy applications on the Stellar network.

The project explores practical use cases of Stellar and Soroban smart contracts — including payments, asset flows, and developer tooling — while maintaining a strong focus on modular architecture, scalability, and contributor-friendly design.

At its core, Orbytech aims to become a reusable foundation for building production-grade Stellar applications with clean abstractions across smart contracts, SDKs, backend services, and frontend interfaces.

## What We Build

- **Smart Contracts (Soroban)**  
  Modular Rust-based contracts for payments, asset transfers, and programmable financial logic on Stellar.

- **Developer SDK**  
  TypeScript-based tools that simplify interacting with Stellar networks and smart contracts.

- **Backend Services**  
  APIs for transaction handling, wallet abstraction, and off-chain coordination logic.

- **Frontend Applications**  
  Modern web interfaces for interacting with Stellar-based applications.

- **Shared Infrastructure**  
  Reusable configs, utilities, and tooling for consistent development across the monorepo.

## Architecture Philosophy

Orbytech follows a **monorepo-first architecture**:

- Modular packages for independent development
- Shared tooling and configuration
- Clear separation between contracts, SDK, backend, and frontend
- Contributor-friendly structure for Wave sprint cycles

## Focus Areas

- Stellar blockchain integration
- Soroban smart contract development
- Developer experience and tooling
- Modular open-source infrastructure
- Scalable Web3 application design

## Contribution Model

We structure work into small, well-scoped issues designed for open-source contributors:

- Bug fixes across frontend, backend, and contracts
- Feature development for payments and wallet flows
- Smart contract enhancements and optimizations
- SDK improvements and developer tooling
- Documentation and onboarding improvements
- Testing and CI reliability

Each issue is designed with clear acceptance criteria to support Wave sprint-style contributions.

## Tech Stack

- **Smart Contracts:** Rust (Soroban SDK)
- **Backend:** Node.js / TypeScript
- **Frontend:** Next.js / React
- **SDK:** TypeScript
- **Blockchain:** Stellar Network

## Goals

- Build a production-ready Stellar development stack
- Lower the barrier to building on Soroban
- Enable fast contributor onboarding through modular design
- Create reusable infrastructure for Web3 applications
